System architecture

Three layers. One clean connection.

Google Sheets is the data layer — the only place student records live. Taskade is the workspace layer — where people act. The HTML boards are the presentation layer — where leadership sees. Each layer does exactly one thing.

Layer 1
🔵 Taskade
Workspace layer. Where people log in, take actions, get notified. Writes to Sheets via webhook. Never stores student data.
Layer 2 — This document
📊 Google Sheets
Data layer. Single source of truth. Every write comes from Taskade or Apps Script. Every read goes to the HTML boards. Lives inside Petersburg's Google Workspace.
Layer 3
🖥️ HTML Boards
Presentation layer. Reads from published Sheets CSV. Renders role-gated views. No login required for board or superintendent audiences.

Twelve steps. Exact order.
Do not skip ahead.

Each step depends on the one before it. Building out of order creates silent errors that only surface when the system is live.

Critical — location matters

Overall status formula

One formula drives every flag
in the system.

Column W in STUDENTS calculates each student's overall compliance status automatically. This is what the reading lead's caseload view shows. This is what drives the flag count in the boards.

STUDENTS col W — paste in W2, drag down
=IF(J2="Pending","Plan not submitted", IF(K2="Pending","Co-ownership pending", IF(M2="Pending","Letter not generated", IF(AND(P2="Pending",ISNUMBER(N2),TODAY()-N2>14),"Parent response — 14+ days", IF(AND(P2="Declined",R2="Required"),"Refusal doc required", IF(U2="Needs Attention","Monitoring: needs attention", IF(V2="Overdue","Monitoring overdue", "On track")))))))
STUDENTS col V — monitoring flag — paste in V2, drag down
=IF(OR(T2="",ISBLANK(T2)),"", IF(TODAY()-T2>SETTINGS!B3+SETTINGS!B4,"Overdue","Clear"))
⚠️
Build these formulas after step 11 — not before. They reference SETTINGS rows B3 and B4 for the monitoring thresholds. If SETTINGS isn't built first, the formula returns a reference error and the entire column breaks.
✅
How to test: After adding at least one test student row, manually set column J to Pending — column W should show "Plan not submitted." Set J to Complete and K to Pending — should show "Co-ownership pending." Work through each condition in order to confirm the chain is correct before connecting Taskade.
